## ЁЯФе What is RightSpeak?

> Most people receive a legal document тАФ an eviction notice, a court summons, a bank notice тАФ and have **no idea what it actually means or what to do next.** RightSpeak fixes that.

RightSpeak uses **Google Gemini 2.5 Flash** to instantly decode any legal document into plain English, giving ordinary people a fair fighting chance against complex legal language.

Upload a PDF, paste text, or even snap a photo of a document. In seconds, you'll get:

- ЁЯУЛ A plain English summary anyone can understand
- ЁЯЪи An urgency rating that tells you how fast to act
- ЁЯУЕ Key facts: dates, deadlines, amounts, parties involved
- ЁЯОп 3 concrete next steps to take right now
- тЪЦя╕П Your legal rights тАФ ones you might not even know you have

## ЁЯЪА Getting Started Locally

### Prerequisites
- Node.js 18+
- A [Google AI Studio](https://aistudio.google.com/) API Key (Gemini)

### 1. Clone the repository

```bash
git clone https://github.com/SavioMohan1/PromptWars.git
cd PromptWars/rightspeak
```

### 2. Install dependencies

```bash
npm install
```

### 3. Configure your API Key

Create a `.env.local` file in the `rightspeak/` directory:

```env
GEMINI_API_KEY=your_gemini_api_key_here
```

### 4. Run the development server

```bash
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) and start analyzing documents! ЁЯОЙ

## тШБя╕П Deploying to Google Cloud Run

From your Google Cloud Shell:

```bash
# Clone the repo
git clone https://github.com/SavioMohan1/PromptWars.git

# Navigate into the app folder
cd PromptWars/rightspeak

# Deploy!
gcloud run deploy rightspeak \
  --source . \
  --region us-central1 \
  --allow-unauthenticated \
  --set-env-vars GEMINI_API_KEY=your_key_here
```

> тЪая╕П **Important:** Replace `your_key_here` with your actual Gemini API key, or configure it in the Google Cloud Console under **Cloud Run тЖТ rightspeak тЖТ Edit & Deploy тЖТ Variables & Secrets**.

## ЁЯФР Security

- The **Gemini API key is never exposed to the client**. All AI calls are made server-side through a secure Next.js API route (`/api/analyze`).
- PDF parsing is done entirely **client-side** тАФ your documents never leave your browser for processing.
- No user data is stored server-side. History is saved locally in the browser's `localStorage`.
